# Для Индиры — Дневник заботы

> Персональное PWA-приложение для беременности: послания, витамины, дневник, фото и забота — всё хранится локально на телефоне.

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/R1M1R/care-diary)

---

## О проекте

**Для Индиры** — одностраничное веб-приложение (PWA), созданное как тёплый цифровой дневник беременности. Работает в браузере и с экрана «Домой» на iPhone как нативное приложение. Записи — в `localStorage`, фото — в **IndexedDB** на устройстве, без сервера и регистрации.

| | |
|---|---|
| **Стек** | HTML · CSS · Vanilla JavaScript |
| **PWA** | Service Worker · Web App Manifest · offline-кэш |
| **Платформа** | iOS Safari (PWA), Android, Desktop |
| **Репозиторий** | [github.com/R1M1R/care-diary](https://github.com/R1M1R/care-diary) |

## Возможности

### Главная
- Персональные приветствия и послания по времени суток
- Обратный отсчёт до даты родов (ПДР)
- Сравнение размера малыша по неделям беременности
- Настроение, вода, дыхательные упражнения
- Фото УЗИ с **автоматическим сжатием**
- Счётчик шевелений малыша
- Быстрый звонок мужу

### Забота
- Расписание препаратов с напоминаниями
- Добавление, редактирование и пауза препаратов
- Отметка приёма и streak-дни

### Дневник
- Письмо / заметки
- График веса и расчёт ИМТ (ВОЗ)
- **Галерея животика** по неделям (19–40): камера + галерея

### Сумка в роддом
- Чек-лист с сохранением прогресса

### Настройки
- Тема (светлая / тёмная), пол малыша, ПДР, рост, телефон
- **Память для фото** — квота, оптимизация, очистка
- Установка PWA на iPhone, проверка обновлений

## Сжатие фото (iOS)

На iPhone 16 Pro и других моделях исходные снимки с камеры могут весить 12–25 MB и **ронять вкладку Safari** при загрузке в память.

Приложение решает это на клиенте **до** сохранения:

| Параметр | УЗИ | Галерея (недели) |
|---|---|---|
| Макс. сторона | 1920 px | 1400 px |
| Макс. вес | ~1.5 MB | ~700 KB |
| Метод | `createImageBitmap` + Canvas JPEG | то же |

Дополнительно:
- HEIC → JPEG через [heic2any](https://github.com/alexcorvi/heic2any) (CDN, по необходимости)
- `URL.revokeObjectURL()` и `ImageBitmap.close()` для освобождения RAM
- Индикатор «Сжимаем фото…» и обработка ошибок через toast
- Автомиграция старых фото из `localStorage` в **IndexedDB** при первом запуске новой версии

## Быстрый старт

### Требования

- Современный браузер (Safari 15+, Chrome, Edge)
- Для PWA и Service Worker — **HTTPS** или `localhost`

### Локальный запуск

```bash
git clone https://github.com/R1M1R/care-diary.git
cd care-diary
npm start
```

Откройте [http://localhost:4173](http://localhost:4173).

> Service Worker активируется только по `http://localhost` или HTTPS — не открывайте файл через `file://`.

## Установка на iPhone (PWA)

1. Откройте приложение в **Safari** по HTTPS.
2. Нажмите **Поделиться** → **На экран «Домой»**.
3. Запускайте с Home Screen — без адресной строки, с splash-экраном и офлайн-кэшем.

Рекомендуется именно PWA: загрузка фото с камеры в отдельном приложении стабильнее, чем во вкладке браузера.

## Деплой

### Vercel (рекомендуется)

1. Импортируйте репозиторий на [vercel.com](https://vercel.com).
2. Root Directory: **`/`** (корень репозитория).
3. Build Command: оставить пустым (статический сайт).
4. Output Directory: **`/`**.

Файл `vercel.json` уже настроен:
- rewrite `/` → `index.html`
- заголовки для `sw.js` и `manifest.webmanifest`

После `git push` в `main` Vercel задеплоит автоматически (если проект подключён).

## Данные и приватность

- Все записи (дневник, лекарства, настройки) хранятся **только в браузере** (`localStorage`).
- **Фото** — в **IndexedDB** (`careDiaryPhotos_v1`). По умолчанию **режим «из галереи»**: в приложении только лёгкое превью (~100 КБ на неделю), оригинал остаётся в галерее iPhone. Опционально можно включить полные копии в настройках.
- Перед сохранением фото **сжимаются** на устройстве (Canvas). Сетка животика подгружается **лениво** — не тормозит при большом количестве снимков.
- Нет аккаунтов, аналитики и отправки данных на сервер.
- «Очистить все данные» в настройках удаляет локальное хранилище и все фото приложения.

## Обновление PWA

При выкладке новой версии:

1. Увеличьте версию кэша в `sw.js`:
   ```js
   const CACHE = 'care-diary-v10'; // было v9
   ```
2. Задеплойте на Vercel.
3. На телефоне: **Настройки → Проверить обновления приложения** или перезапустите PWA.

## Разработка

| Задача | Действие |
|---|---|
| Локальный сервер | `npm start` |
| Перегенерировать иконки | `npm run icons` (только сжатие, дизайн не меняется) |
| Проверка SW | DevTools → Application → Service Workers |
| Квота storage | Настройки → Память для фото |
| Сброс данных | Настройки → Очистить все данные |

### Альтернатива для сжатия (npm)

Текущая реализация — **нативный Canvas** без сборки. При переходе на bundler можно использовать:

```bash
npm install browser-image-compression
```

### OneDrive / облачная синхронизация

> **Важно:** OneDrive может подменить `apple-icon.png` старой тяжёлой версией (~3.6 MB). Не заменяй файл вручную — восстанови из git:

```bash
git restore apple-icon.png
```

Если нужно пересжать **тот же** дизайн после замены оригинала:
```bash
npm run icons
```

CI проверяет, что `apple-icon.png` не больше 500 KB и 512×512 px.
